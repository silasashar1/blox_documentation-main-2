 Step 1: Access APIs & Services

Navigate to the **APIs & Services** section within your GCP project.

![APIs and Services screen](https://d1r1e7xjkfj7nz.cloudfront.net/image-20240422-100541.png "APIs and Services")
 
 Step 2: Select Credentials

In the APIs & Services dashboard, click on the **Credentials** tab.

![Credentials screen](https://d1r1e7xjkfj7nz.cloudfront.net/image-20240422-100644.png "Credentials")

 Step 3: Create an OAuth Client ID

Create an OAuth Client ID credentials. Choose **Web application** as the application type.

![OAuth Client ID creation screen](https://d1r1e7xjkfj7nz.cloudfront.net/image-20240422-100901.png "OAuth Client ID Creation")

### Important Details

- The created application will provide you with a **Client ID** and **Client Secret**. These are crucial for your application's authentication processes.

![Client ID and Secret](https://d1r1e7xjkfj7nz.cloudfront.net/image-20240422-101523.png "Client ID and Secret")

 Step 4: Configure Redirect URIs

Paste the Redirect URIs generated into the **Authorized redirect URIs** section in your application settings.

![Setting Redirect URIs](https://d1r1e7xjkfj7nz.cloudfront.net/image-20240430-111507.png "Redirect URIs")

For more detailed instructions and visuals, visit [this link](https://d1r1e7xjkfj7nz.cloudfront.net/).

Make sure to save your configuration after making these changes.
